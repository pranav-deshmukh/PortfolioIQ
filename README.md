# PortfolioIQ

**PortfolioIQ** is an **AI-powered portfolio intelligence platform** that ingests real-time financial news, converts it into **structured market signals** using **custom ML models**, quantifies impact on **client portfolios** with risk analytics (VaR/CVaR/volatility/beta/drawdown/concentration), and then uses an **LLM agent with tool-calling** to generate **advisor-ready alerts and explanations**.

> Built for the “advisor copilot” use case: **quant-driven, explainable, and actionable**—not “LLM predicts markets.”

---

## Why PortfolioIQ?

Wealth advisors and portfolio managers don’t just need news—they need answers:

- **What changed?**
- **Which client portfolios are exposed?**
- **How big is the likely impact (in % and $)?**
- **Is this noise or a regime shift?**
- **What should an advisor communicate to the client?**

PortfolioIQ is designed to turn unstructured headlines into **portfolio-specific, quantified insights** and **proactive alerts**.

## Key Components

### 1) AI Agent (`agent/`)
The autonomous “brain” that:
- ingests and deduplicates news,
- runs analytics and ML inference,
- calls tools,
- generates prioritized alerts + recommendations.

### 2) Analytics Engine (`agent/analytics_engine.js`)
The quantitative core that performs:
- event → sector impact modeling,
- portfolio impact mapping to holdings,
- risk metrics (VaR/CVaR/volatility/beta/drawdown),
- aggregation + normalization to avoid unrealistic compounding,
- alert candidate detection based on thresholds and confidence.

### 3) Backend API (`Server/`)
Express + TypeScript API exposing endpoints for:
- portfolios & holdings,
- news feed,
- risk metrics,
- macro indicators,
- performance,
- data synchronization.

External integrations referenced in the project include:
- market/news providers (e.g., Twelve Data, Finnhub, NewsData),
- macro data (e.g., FRED),
- LLM routing (e.g., OpenRouter),
- persistence via MongoDB Atlas.

### 4) Dashboard (`dashboard/`)
Next.js + TypeScript + TailwindCSS frontend for:
- portfolio overview,
- real-time news with impact annotations,
- risk dashboards (VaR, volatility, concentration),
- alert workflows,
- advisor chat interface to query the AI agent.

### 5) ML Notebooks (`ml_notebooks/`)
Training artifacts for three deployed ML models (HuggingFace Spaces style):
- **Event Impact Classifier** (news → sector impacts + confidence)
- **Portfolio Risk Scorer** (allocation → risk score + category + drivers)
- **Volatility / VaR Model** (forecast volatility + VaR-style downside metrics)

## Memory Manager (How the Agent Gets “Memory”)

PortfolioIQ uses a lightweight **persisted memory snapshot** per client. Instead of re-fetching everything on every chat turn, the agent can load a single Markdown “Client Memory” document and treat it as the default source of truth.

### Where memory is stored
- `agent/memory/memory_<CLIENT_ID>.md`  
  Example: `agent/memory/memory_C004.md`

### How memory is created
Generated by `agent/memory_manager.js` → `createClientMemory(clientId)`.

It builds a structured Markdown report by gathering (in parallel):
- Risk metrics via `computeVaR()` (VaR/CVaR/volatility/beta/drawdown/concentration)
- Active alerts
- Latest advisor insights
- Recent analytics snapshots (stress tests / Monte Carlo when available)
- Recent news context (from MongoDB `news_events`)

Then it writes the snapshot into `agent/memory/`.

### How memory is used in chat
- `loadClientMemory(clientId)` reads the latest snapshot file.
- The chat server injects that snapshot into the LLM prompt as a **system message** (when present), so the agent uses it as the baseline context for that client.

## The 3 ML Layers (Explainable by Design)

PortfolioIQ uses layered ML so the LLM doesn’t “guess”—it **explains deterministic outputs**.

1. **Event Impact Classifier**  
   Converts headlines/descriptions into predicted impacts across sectors:
   `tech, financials, energy, healthcare, bonds, commodities, international`  
   plus a **confidence** score.

2. **Portfolio Risk Scorer**  
   Converts a client’s allocation into an **explainable risk score** + category, using concentration/diversification style features.

3. **Volatility / VaR Forecasting**  
   Uses market-statistical modeling (e.g., GARCH-style logic) to estimate forward risk:
   volatility, VaR, CVaR, stressed scenarios.

## End-to-End Flow

```
1) News ingestion
   External APIs → dedupe/normalize → store

2) Analytics
   News → sector impacts (ML + fallback rules)
        → regime-aware aggregation & caps
        → per-client portfolio impact
        → VaR/CVaR/volatility/beta/drawdown/concentration

3) Agent reasoning (LLM + tools)
   Structured analytics JSON → decide alert/monitor/ignore
                            → draft advisor-ready explanations/actions

4) Dashboard
   Advisors review alerts, metrics, and chat with the system
```

## What Makes It “Agentic”? 

A real copilot should not only chat; it should:
- plan,
- call tools,
- verify quantitative results,
- generate structured outputs,
- and produce a client-ready narrative.

PortfolioIQ’s approach is explicitly **hybrid**: **quant + ML signals first**, **LLM explanation second**.

## Tech Stack

- **Frontend:** Next.js, TypeScript, TailwindCSS  
- **Backend:** Express.js, TypeScript  
- **Agent Runtime:** Node.js (agent loop + tool calling)  
- **ML:** Python, scikit-learn / XGBoost-style models, GARCH-style volatility modeling  
- **DB:** MongoDB Atlas  
- **Deployment Pattern:** ML models deployed as Gradio APIs (HuggingFace Spaces pattern)
